# /daily — 일일 회고 풀파이프라인

사용자 입력: `$ARGUMENTS`

다음 순서로 진행한다. 각 단계 출력은 다음 단계의 입력으로 그대로 전달한다.

## 0단계 — 캘린더 맥락 수집
`calendar-context` 스킬을 호출해 오늘 일정 / 내일 일정 / 내일 빈 슬롯을 가져온다.
- 캘린더 권한 없음/오류면 "캘린더 없이 진행" 한 줄 남기고 1단계로.
- 사용자가 "캘린더 빼" 라고 명시하면 이 단계 건너뛴다.

## 1단계 — 입력 정리
사용자 입력이 자유서술이면 `daily-log` 스킬의 표준 포맷으로 변환한다.
입력이 이미 정리된 형태라면 이 단계는 생략한다.

## 2단계 — 회고
`daily-summary` 서브에이전트를 호출해 (성과 3줄 / 걸리는 것 2가지 / 내일 1순위 1가지) 출력을 받는다.
0단계의 캘린더 맥락이 있으면 입력과 함께 묶어 전달한다.

## 3단계 — 블로커 해결안
2단계에서 나온 "걸리는 것 2가지"를 입력으로 `blocker-resolver` 서브에이전트를 호출한다.

## 4단계 — 내일 1순위 분해
2단계에서 나온 "내일 1순위 1가지"를 입력으로 `task-breaker` 서브에이전트를 호출한다.
0단계의 "내일 일정 + 빈 슬롯"이 있으면 같이 넘긴다 → 표의 "예정 슬롯" 열이 채워진다.

만약 2단계에서 1순위 후보가 단일하지 않다고 표시되면, `task-breaker` 호출 전에 `next-day-priority` 스킬로 1순위를 먼저 확정한다.

## 5단계 — 캘린더 등록 (선택)
사용자가 "캘린더에 박아줘" / "내일 일정 등록해줘" 라고 명시한 경우에만 `schedule-writer` 서브에이전트를 호출한다.
- 먼저 드라이런만 보여주고 사용자 "ok" 받은 다음에 실제 `create_event` 호출.
- 사용자가 시간을 명시하지 않으면 0단계의 빈 슬롯 중 가장 빠른 곳을 시작점으로.

## 6단계 — 저장 (선택)
사용자가 "저장해" / "기록해" 라고 말한 경우에만 `~/.claude-daily/tomorrow.md` 에 4단계의 작업과 첫 30분 단계만 1줄로 기록한다. 디렉터리가 없으면 만든다.

## 출력 규칙
- 모든 출력은 한국어.
- 각 단계 시작 전에 어떤 에이전트/스킬을 호출하는지 한 줄로 표시.
- 사용자가 "요약만", "블로커만" 등 부분만 요청한 경우 해당 단계까지만 실행.
