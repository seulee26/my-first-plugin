# 오늘 한 일 표준 로그 작성 스킬

## 목적
회고 에이전트가 일관되게 처리할 수 있도록, 자유서술 형태의 하루 기록을 다음 포맷으로 정리한다.

## 출력 포맷
모든 출력은 **한국어**.

```
## 오늘 한 일 (YYYY-MM-DD)

### 만든 것 / 진전
- [영역] 무엇을 어디까지

### 회의 / 커뮤니케이션
- [상대] 안건 → 결론

### 의사결정
- 무엇을 결정했고, 근거는

### 미해결 / 다음으로 넘긴 것
- 항목 — 이유

### 시간 외 (선택)
- 학습, 휴식, 잡일 등 회고 가치가 있는 것만
```

## 작성 규칙
- 활동이 아니라 **산출물 / 진전 / 결론** 중심으로 적는다. ("회의함" ❌ → "스펙 v2로 합의" ✅)
- 사용자가 말하지 않은 항목은 비워둔다. 추측해서 채우지 않는다.
- 정량 정보(숫자, 진척률, 시간)가 있으면 우선 보존한다.
- 비공개 정보(이름, 금액 등)도 사용자가 입력한 그대로 유지. 임의로 마스킹하지 않는다.
- 한 항목이 여러 영역에 걸치면 가장 결과가 큰 영역에만 적고 다른 곳에 중복 작성하지 않는다.

## 사용 흐름
1. 사용자가 던진 자유서술을 위 포맷으로 변환한다.
2. 변환 결과만 출력한다 (해설·평가 금지).
3. 이 출력은 보통 그대로 `daily-summary` 서브에이전트의 입력으로 전달된다.
